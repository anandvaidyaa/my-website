Cloud Serverless Architecture & Live Telemetry

What it does: Incoming orders from client apps hit the Azure APIM gateway for authentication and rate-limiting. Once validated, they trigger an Azure Function running serverless business logic (syncing inventory and calculating shipping rates) before persisting state to Cosmos DB.

Why this design: This decoupled, event-driven pattern ensures low execution costs by scaling compute dynamically to zero, protects database limits, and ensures zero order loss even during sudden traffic surges.
